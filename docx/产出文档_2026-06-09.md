# 产出文档 — 2026-06-09

## 概述

今日主要完成了 GrayVideoDL 的核心功能开发，包括视频下载功能、B站登录认证、设置页面、独立日志页面、数据模型建立以及函数注册表自动化工具。

## 一、新增功能模块

### 1. B站登录认证

- **`BilibiliLoginFragment.java`** — 全屏 WebView 登录页（DialogFragment 实现），支持二维码扫码登录和密码登录
  - 内置加载进度条，自动检测登录状态
  - 通过 JavaScript 注入读取 `document.cookie`，检测 `SESSDATA` 判定登录成功
  - 登录后自动提取 Cookie 并保存为 Netscape 格式供 yt-dlp 使用
  - 修正了键盘输入时页面无法滚动的问题（`SOFT_INPUT_ADJUST_RESIZE`）

- **`BilibiliLoginDialog.java`** — 旧版登录对话框（保留，通过 CookieManager 提取 Cookie）
  - 提供 `getCookieFilePath()` 和 `isLoggedIn()` 静态方法供 HomeFragment 调用
  - `loadFromAssets()` 支持从 assets 预置 Cookie 文件

- **`fragment_bilibili_login.xml`** — 登录页面布局（WebView + 加载进度条 + 关闭按钮）

### 2. 设置页面（SettingsFragment）

- **`SettingsFragment.java`** — 完整的设置页面
  - 默认画质选择：自动 / 4K / 2K / 1080p / 720p / 480p / 360p
  - 调试日志开关：控制底部导航栏日志 Tab 显隐
  - B站登录状态展示：已登录/未登录，点击可登录或退出
  - 退出登录时清除 SharedPreferences、Cookie 文件和 WebView 缓存
  - 所有设置通过 SharedPreferences 持久化（`grayvideodl_settings`）

- **`fragment_settings.xml`** — 设置页面布局（卡片式设置项列表）

### 3. 视频下载功能（HomeFragment v2）

- **HomeFragment.java 重写** — 完整的视频解析和下载流程
  - **自动合并音视频开关**：与设置页面联动，合并模式下仅显示视频格式
  - **格式分栏显示**：视频画质 / 音频 两个独立区域
  - **画质警告横幅**：默认画质不满足时内联提示（替代模态框）
  - **成功动画提示**：解析成功后顶部渐显 "✓ 解析成功"，5秒后渐隐
  - **重复解析检测**：相同 URL 二次解析时弹出确认对话框
  - **自动格式选择**：根据默认画质设置自动选中最匹配的格式
  - **下载功能**：调用 Python yt-dlp 下载，完成后通过 MediaStore API 复制到公共 Download 目录
  - **格式锁定标记**：会员格式显示 "🔒 需大会员" 并置灰不可点击

### 4. 独立日志页面（LogFragment + LogBuffer）

- **`LogFragment.java`** — 独立的调试日志展示页面
  - 从 LogBuffer 读取日志，自动滚动到底部
  - 支持复制到剪贴板和清空日志
  - 页面切换自动刷新（`onHiddenChanged` 回调）

- **`LogBuffer.java`** — 日志缓冲区单例
  - HomeFragment 写入 → LogFragment 读取
  - 跨 Fragment 共享日志数据

- **`fragment_log.xml`** — 日志页面布局

### 5. 数据模型层

- **`VideoInfo.java`** — 视频信息数据模型
  - 解析 yt-dlp 返回的 JSON 数据
  - 内置 `Format` 子类：格式 ID、扩展名、分辨率、文件大小、编码信息、锁定状态
  - `resolutionToQuality()` 方法将 "1920x1080" 转换为 "1080p"
  - `isAudioOnly()` / `isLocked()` 判断方法

- **`DownloadTask.java`** — 下载任务数据模型

  - 持久化到应用内部存储（`download_tasks.json`）
  - 状态管理：downloading / completed / failed
  - 文件大小格式化、时间格式化

### 6. MainActivity 改造

- **`MainActivity.java`** — 主 Activity 重写
  - 4个 Tab 页底部导航：首页 / 下载 / 日志 / 设置

## 四、技术要点

1. **Chaquopy Python 桥接**：通过 `Python.getInstance().getModule("ytdlp_bridge")` 在 Android 中调用 Python 代码
2. **Cookie 传递**：B站登录后提取 Cookie → Netscape 格式保存 → 传递给 yt-dlp 的 `cookiefile` 参数
3. **MediaStore API**：Android 11+ 文件写入公共目录的标准方式，使文件在文件管理器中可见
4. **格式锁定检测**：解析 yt-dlp 返回的 `format_note`，判断是否存在 "missing"/"premium"/"会员" 关键词
5. **Fragment 懒加载**：使用 `hide/show` 机制替代 `replace`，避免 Fragment 重复创建

## v0.3A 更新内容

### 1. B站登录检测修复（核心修复）

#### 问题根因

B站移动端 H5 登录流程中，SESSDATA **不通过 Set-Cookie 响应头设置**，而是通过**跨域重定向 URL 参数**传递：
1. 用户在 `passport.bilibili.com/login` 输入账号密码
2. 登录成功后，B站 服务端将 SESSDATA、bili_jct、DedeUserID 等凭证放在 URL 参数中，依次传递给子域（biligame → bilibili.cn → huasheng.cn）
3. 各子域服务器从 URL 参数获取凭证并设置对应域的 Cookie

因此 `CookieManager.getCookie(".bilibili.com")` 永远拿不到 SESSDATA，原有检测机制无效。

#### 修复方案

新增 `extractCookiesFromUrl()` 方法，在 `shouldOverrideUrlLoading()` 中拦截跨域 URL（`contains("crossDomain") && contains("SESSDATA=")`），从 URL 查询参数中手动提取凭证。

#### 涉及文件
| 文件 | 说明 |
|------|------|
| `BilibiliLoginFragment.java` | 新增 `extractCookiesFromUrl()` 方法 + 新增调试日志 |
| `SettingsFragment.java` | 新增登录流程调试日志 |

### 2. 调试日志体系

在两个核心类中添加了完整的 Logcat 调试日志，便于后续排查登录相关问题：

- **`BilibiliLoginFragment.java`** — TAG=`BiliLogin`，覆盖 Cookie 清除、页面加载、导航拦截、Cookie 检测、文件保存全流程
- **`SettingsFragment.java`** — TAG=`SettingsFrag`，覆盖登录状态切换、登录回调、退出登录全流程
